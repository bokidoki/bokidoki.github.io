软件更新透明度要求详解

概述

软件更新透明度是 PSTI 法规三项核心安全要求之一。制造商必须向消费者明确声明产品安全更新的支持期限。本文详细解读 ETSI EN 303 645 条款 5.3 的技术要求及合规实现。

一、法规要求

1.1 ETSI EN 303 645 条款 5.3

原文：

When software updates are made available to consumers, they should be informed about the security implications of the update, and the update should be easy to install.

核心要点：

要点 要求
信息告知 通知用户更新的安全意义
安装便捷 更新应当易于安装

1.2 PSTI Regulations 2023 具体要求

制造商必须公开声明：

内容 要求
支持期限 明确安全更新支持的截止日期
获取方式 说明如何获取安全更新
通知方式 说明如何通知用户

二、安全更新支持期限声明

2.1 期限确定因素

因素 说明
产品生命周期 产品预期的市场生命周期
技术可行性 维护更新的技术能力
商业策略 企业产品迭代计划
行业惯例 同类产品的支持期限

2.2 建议期限

产品类型 建议支持期限
智能手机 3-5年
智能家居设备 3-5年
智能安防设备 5年以上
智能门锁 5年以上

手动下载 所有设备 用户可控 用户可能忽略
App更新 有配套App 便捷、可通知 需安装App

方案二：手动更新

要素 内容
版本号 更新版本标识

安全影响 更新修复的安全问题
安装说明 如何安装更新

四、用户通知机制

4.1 通知方式

方式 说明 适用场景
设备推送 设备端弹窗通知 联网设备
邮件通知 发送邮件 已注册用户
App通知 推送通知 有配套App
官网公告 网站发布 所有用户
社交媒体 社交平台发布 大范围通知

4.2 通知内容

要素 内容
更新摘要 更新内容概述
安全影响 更新的安全意义
操作指引 如何获取和安装
截止日期 建议更新时间

4.3 通知示例

五、支持结束处理

5.1 提前通知

在支持结束前，应提前通知用户：

时间点 行动
结束前6个月 发送首次通知
结束前3个月 发送提醒通知
结束前1个月 发送最终通知

5.2 结束后建议

建议 说明
升级换代 建议购买新型号
停用敏感功能 停止使用敏感网络功能
网络隔离 将设备隔离在不安全网络

5.3 结束通知示例

六、合规检查清单

6.1 声明准备

  • 确定安全更新支持期限
  • 编写支持期限声明
  • 说明更新获取方式
  • 说明用户通知方式
  • 在合规声明中包含上述信息

6.2 更新机制

  • 建立更新发布流程
  • 建立更新通知机制
  • 建立更新安装验证
  • 建立更新问题处理流程

6.3 用户沟通

  • 建立用户通知渠道
  • 准备更新说明模板
  • 准备安全公告模板
  • 建立客服培训

小结

软件更新透明度要求核心要点：

  1. 明确期限：声明安全更新支持截止日期
  2. 说明方式：告知用户如何获取更新
  3. 及时通知：建立用户通知机制
  4. 结束告知：提前告知支持结束
