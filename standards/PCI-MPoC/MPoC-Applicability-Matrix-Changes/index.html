PCI MPoC v1.1 适用性矩阵变更对各解决方案类型的影响分析

概述

PCI MPoC v1.1 对 MPoC Requirements Applicability Matrix（要求适用性矩阵） 进行了重大重构。本文基于 MPoC v1.1 标准、v1.0.1 标准以及 Summary of Changes 文档，详细分析此次变更对各类 MPoC 解决方案的影响。

一、适用性矩阵的结构性变化

1.1 评估模型转变

维度 v1.0.1 v1.1
定义基础 基于角色（谁负责执行） 基于产品/解决方案（什么被评估）
关注点 实体职责 产品类型与评估范围

1.2 解决方案类型对比

v1.0.1 列定义（基于角色）：

  • MPoC Solution Provider
  • MPoC Software Vendor
  • A&M Service Provider
  • Monolithic MPoC Solution
  • Using Listed MPoC Software
  • Using Listed MPoC Software and A&M Service

v1.1 列定义（基于产品/解决方案）：

  • MPoC Software - 独立软件产品
  • MPoC Service - 独立服务产品（细分为 A&M、A&M & Payment Processing）
  • MPoC Solution Integrating MPoC Software - 集成软件的解决方案
  • MPoC Solution Integrating A&M Service - 集成 A&M 服务的解决方案
  • MPoC Solution Integrating A&M and Payment Service - 集成 A&M 和支付服务的解决方案
  • Monolithic MPoC Solution - 单体解决方案

二、Domain 适用性变更

2.1 Domain 适用性对比表

Domain v1.0.1 适用对象 v1.1 适用对象 变化说明
Domain 1 Software Vendor, Monolithic Solution MPoC Software, MPoC Service, Monolithic Solution 扩展至 MPoC Service 产品
Domain 2 Solution Provider (Using Listed Software), Monolithic Solution MPoC Solution (集成类), Monolithic Solution 逻辑一致，定义更清晰
Domain 3 A&M Service Provider, Solution Providers MPoC Solution (集成 Service) 重大转变：Service 产品本身不再适用
Domain 4 Vendor, Service Provider, Solution Provider MPoC Software, MPoC Service, MPoC Solution 广泛适用
Domain 5 Solution Provider MPoC Solution 明确限制为解决方案层面

2.2 关键变化解读

Domain 3 适用性转移：

v1.0.1 中，A&M Service Provider 需满足 Domain 3 要求。

v1.1 中，MPoC Service 产品本身不适用 Domain 3，而是由集成该服务的 Solution Provider 来满足。这体现了”产品评估”与”运营评估”的分离。

三、各解决方案类型影响详解

3.1 MPoC Software

影响项 说明
术语统一 引入 “COTS-based MPoC Software” 统称 MPoC SDK 或 MPoC Application
SDK 集成能力 新增 MPoC SDK 可集成另一 MPoC SDK 的能力（最多一个）
新增要求 SR 1A-1.5：验证不传递敏感资产给其他 SDK/Application
新增要求 SR 1A-1.6：SDK 集成场景验证
新增要求 SR 1G-1.12：供应商验证机制
适用范围 Domain 1, 4（Domain 2, 3, 5 不适用于独立软件产品）

3.2 MPoC Service

影响项 说明
名称变更 从 “Attestation and Monitoring Service MPoC Product” 变更为 “MPoC Service”
产品类型确立 首次作为独立产品类型纳入评估框架
细分类型 A&M Service、A&M & Payment Processing Service
Domain 1 适用 必须集成现有 MPoC Software 或满足独立 MPoC Software 要求（注 8）
Domain 3 不适用 产品本身不适用 Domain 3，由集成方负责
适用范围 Domain 1, 4

3.3 MPoC Solution Integrating MPoC Software

影响项 说明
架构变化 支持一个 MPoC SDK 集成另一 MPoC SDK
指导要求 SR 1G-1.7：指导需足以支持安全集成
新增要求 SR 1G-1.8：安全通道管理指导验证
适用范围 Domain 2, 3, 4, 5

3.4 MPoC Solution Integrating A&M Service

影响项 说明
Domain 3 责任 Solution Provider 承担 Domain 3 运营要求
产品评估分离 A&M Service 产品已单独评估，Solution Provider 需验证集成合规性
适用范围 Domain 2, 3, 4, 5

3.5 MPoC Solution Integrating A&M and Payment Service

影起项 说明
最复杂场景 集成 A&M 服务和支付处理服务
Domain 3 责任 同样由 Solution Provider 承担
支付处理要求 需满足 Domain 4 支付处理相关要求
适用范围 Domain 2, 3, 4, 5

3.6 Monolithic MPoC Solution

影响项 说明
Module 1G 豁免 明确不需要满足集成指南要求（SR 1G）
原因 单体方案不需要集成第三方组件
适用范围 Domain 1, 2, 3, 4, 5（全部适用）

四、Module 结构性调整

4.1 Module 3B 位置移动

项目 v1.0.1 v1.1
模块名称 COTS Platform Baseline and Vulnerability Management COTS Baseline and Vulnerability Management
所属 Domain Domain 3 (Attestation and Monitoring) Domain 4 (Software Management)
新编号 Module 3B Module 4A-3

意义：基线管理从”证明与监控”域移至”软件管理”域，体现将其视为软件生命周期管理的一部分。

4.2 Domain 3 模块重组

v1.0.1 v1.1 说明
Module 3A Module 3A 保持
Module 3B → Module 4A-3 移至 Domain 4
Module 3C Module 3B 重新编号
Module 3D Module 3C 重新编号为 Operational Security

五、其他适用性相关变更

5.1 PCI PTS POI 适用范围扩大

变更 说明
v1.0.1 仅适用于 PCI PTS POI SCRP 批准类别
v1.1 扩展至所有 PCI PTS POI 设备

影响需求：SR 1D-2.3, SR 1E-1.9 等

5.2 后端系统范围澄清

需求 澄清内容
SR 1B-1 Preamble 适用于 COTS 设备上执行的代码，不适用于后端系统
SR 1B-2 Preamble 本节不适用于后端系统

5.3 敏感数据范围澄清

需求 澄清内容
SR 1D-1 Preamble 按 PCI DSS FAQ 截断的 PAN 不在本需求范围内
SR 1D-5 Preamble 同上

5.4 PCI DSS DESV 引用移除

需求 变更
SR 3D-1.1 移除 PCI DSS DESV 验证要求，保留 PCI DSS 验证要求

六、变更影响总结

6.1 按解决方案类型汇总

解决方案类型 主要影响 需关注的新要求
MPoC Software SDK 集成能力、术语统一 SR 1A-1.5, 1A-1.6, 1G-1.12
MPoC Service 产品类型确立、Domain 3 豁免 注 8 条件
Solution (Software) 架构灵活性增加 SR 1G-1.7, 1G-1.8
Solution (A&M Service) Domain 3 责任明确 运营合规验证
Solution (A&M & Payment) 最复杂场景 全 Domain 适用
Monolithic Solution Module 1G 豁免 无新增

6.2 按变更性质汇总

变更性质 影响范围
结构性 适用性矩阵重构、Module 3B 移动
范围扩大 PCI PTS POI 支持范围
范围澄清 后端系统、截断 PAN
新增能力 SDK 集成 SDK
责任转移 Domain 3 从 Service Provider 到 Solution Provider
